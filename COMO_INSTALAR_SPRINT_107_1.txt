FARPHA — SPRINT 107.1
ESTABILIZAÇÃO DA EXPERIÊNCIA PÚBLICA, AUTENTICAÇÃO E AJUDA

ANTES DE COMEÇAR
1. No terminal onde npm run dev está aberto, pressione Ctrl+C.
2. Confirme que o ZIP está em Downloads.
3. Não apague a pasta AgriOS e não execute SQL para esta atualização.

INSTALAÇÃO — BLOCO ÚNICO PARA GIT BASH

cd ~/Documents/Projetos

BACKUP="AgriOS_backup_sprint107_1_$(date +%Y%m%d_%H%M%S)"
mkdir -p "$BACKUP/frontend"

cp -a AgriOS/frontend/src "$BACKUP/frontend/"
cp -a AgriOS/frontend/public "$BACKUP/frontend/"
cp -a AgriOS/frontend/scripts "$BACKUP/frontend/"

for ITEM in package.json package-lock.json vite.config.ts tsconfig.json tsconfig.app.json tsconfig.node.json index.html eslint.config.js .nvmrc; do
  if [ -e "AgriOS/frontend/$ITEM" ]; then
    cp -a "AgriOS/frontend/$ITEM" "$BACKUP/frontend/"
  fi
done

unzip -o "$HOME/Downloads/FARPHA_Sprint_107_1_Experiencia_Estabilizada.zip" -d "$HOME/Documents/Projetos"

cd ~/Documents/Projetos/AgriOS/frontend
npm install
npm run validate
npm run dev

O ficheiro frontend/.env.local não é substituído pelo ZIP.

CONFIRMAÇÃO VISUAL
- Na página inicial, a instalação aparece apenas na barra de navegação quando o navegador permite instalar.
- O Centro de Ajuda aparece uma única vez.
- O Centro de Ajuda possui Inteligência, Pedidos e Equipa.
- Entrar e Criar conta possuem rolagem própria e não exigem F11.
- A marca FARPHA utiliza o componente vetorial verde.
- No telemóvel, o botão de ajuda fica acima da navegação inferior.

GOOGLE E MICROSOFT — PASSO OBRIGATÓRIO NO SUPABASE
O frontend deixa de abrir a página JSON de erro e apresenta uma mensagem amigável quando o provedor não está ativo. Para o login funcionar de verdade:

1. Abra o mesmo projeto Supabase indicado por VITE_SUPABASE_URL.
2. Entre em Authentication > Providers.
3. Google: ative Enable Sign in with Google, introduza Client ID e Client Secret e guarde.
4. Azure: ative Enable Sign in with Azure, introduza Client ID e Client Secret e guarde.
5. Confirme as Callback URLs nos painéis Google Cloud e Microsoft Entra.
6. Em Authentication > URL Configuration, mantenha http://localhost:5173 e http://localhost:5173/** durante o desenvolvimento.
